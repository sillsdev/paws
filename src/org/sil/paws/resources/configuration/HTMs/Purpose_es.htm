Declaración del propósito del programa PAWS

Como el nombre del programa indica, PAWS es un programa de Análisis y tutorial para autores de sintaxis y entonces tiene dos propósitos básicos y dos clases de salidas:

  • El tutorial para autores elabora un borrador de por lo menos una clase de descripción de la gramática de la sintaxis de su lengua, basado en las respuestas a las preguntas en cada sección de PAWS. Este archivo de salida está en formato de página web y en el formato de XML para el programa de XLingPaper. (La documentación paraXLingPaper se incluye en la sección de los recursos del menú de ayuda. Véase también Simons and Black (2009) y Black (2009) para ver porqué este es un buen formato a utilizar.)
  • El programa de análisis produce todos los archivos necesarios para utilizar el programa de análisis sintáctico de PC-PATR (McConnel 2002, incluido en la sección de los recursos del menú de ayuda) para hacer la desambiguación dentro de CARLAStudio.

Los que desean utilizar el tutorial para autores solamente, estén seguros de no presionar donde dice el "Producir el archivo de la gramática de PC-PATR y los archivos de ejemplos para probar" en la sección de las opciones de salidas. (Esto también ocultará todas las instrucciones en rojo en cada sección que se relacionan con la anotación de su léxico con las características necesarias para utilizar el programa de análisis de PC-PATR.) Una vez que se terminan todas las secciones, los archivos de salida del tutorial para autores pueden ser corregidos y pulidos. (Si su interés está solamente en el tutorial para autores, usted puede ahora volver a la página de contenido. Vuelva al Contenido) Los que desean hacer el análisis también obtendrán por lo menos una descripción de la gramática de manera adicional.

El propósito del programa de análisis es ayudar al usuario a determinar las reglas iniciales de la estructura de frase, de las entradas léxicas, de las plantillas de las características y de las restricciones necesarias para utilizar PC-PATR. Esas características de la lengua que son generales, generalmente cerca de 3/4 de las reglas, se pueden describir por medio de este programa. Las reglas específicas a una lengua necesitarán ser agregadas más adelante utilizando las herramientas del interfaz que se proporcionarán para ayudar al usuario y/o ayudar al consultor.

Establecer un análisis sintáctico completo para una lengua es una tarea monumental. La experimentación por lo tanto fue hecha para determinar cuánto era necesario alcanzar la cobertura adecuada de la lengua para que el programa de análisis pueda quitar ambigüedades de salida de AMPLE dentro de CARLAStudio. En la escala de desplazamiento entre el más simple y el más complejo, este programa de análisis podríamos decir que se ubica en el centro. Específicamente, es más simple en cierto modo y más complejo en otros que el análisis presentado dentro el texto de Bickford (1998) Tools for Analyzing the World's Languages: Morphology and Syntax (que se utiliza en muchos de los cursos de SIL Gramática I). Las diferencias principales son:

  • Se utiliza la subcategorización solamente muy limitada y parcial, puesto que éste es el punto que requiere la mayoría del tiempo y de la complejidad en un programa de análisis completo.
  • Una estructura de frase articulada conforme a la teoría de la X-barra se utiliza ( véase Black 1999, incluido en la sección de los recursos del menú de ayuda, para introducción), puesto que las estructuras planas con las porciones opcionales son mucho más difíciles para restingir correctamente y requerir más memoria y tiempo de transformación.
  • Un programa de análisis trabaja necesariamente en la estructura superficial, así que las reglas de la estructura de frase deben mostrar la salida de transformaciones, no la estructura subyacente. Este programa de análisis no incluye los rastros u otros elementos nulos que serían necesarios en un programa de análisis completo (véase Black 1997 para ejemplo; se incluye en la sección de los recursos del menú de ayuda).

Este programa está diseñado por secciones, trabajando desde las frases más pequeñas que se pueden incluir en otras frases hasta las frases y las cláusulas más complejas. Esto permite que el usuario trabaje en una sección a la vez, que pruebe las hipótesis para cada tipo de frase y al mismo tiempo llegue a familiarizarse utilizando PC-PATR para la desambiguación dentro de CARLAStudio. Sin embargo el programa entero debe terminarse antes de analizar a los textos regulares.
